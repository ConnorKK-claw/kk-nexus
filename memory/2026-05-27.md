# 2026-05-27 关键决策与教训

## 关键决策
1. **GBrain 集成到 Super KAM 体系**：作为语义检索引擎，不替代 KAM vault
2. **全部数据导入**（含 raw/）：GBrain 的 RRF 混合检索天然能处理噪声，无需预先过滤
3. **001 优先改造**：equity-incentive 知识库全部导入，002-004 暂缓
4. **--batch-size 3**：解决 nomic-embed-text 在 Iris Xe 集显上卡死的关键参数

## 技术教训
1. nomic-embed-text 在 CPU 推理下约 3-4s/chunk，全量 6632 chunks 约 1.5 小时
2. GBrain embed 有 30min wall-clock budget 硬限制，需要分多次运行
3. ollama 进程容易在长时间负载后退出，需要保持监控
4. 超长文档（>8192 tokens）会被 nomic-embed-text 拒绝，约 0.3% 的 chunks 受影响

## 架构决策
- Super KAM = 行为规范 + WAL 协议 + 蒸馏流程（不可替代）
- GBrain = 语义检索 + 合成回答（补充层）
- llm-wiki = 网页摄取 + 非结构化内容（数据源层）
- 三层互补，不替代
