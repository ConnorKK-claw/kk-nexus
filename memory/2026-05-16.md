## [12:00] 控制字符清理导致文件损坏

**发生了什么:**
使用 PowerShell [string].Replace([char]12, '') 清理 raw/user 文件中的 form feed 控制字符。
''（空字符串）不能转换为 [char] 类型，方法调用抛出异常。
异常后 Set-Content 写入空值，12 个 raw/user 文件被刷成 3 字节空文件。

**怎么处理的:**
1. 删除损坏的 3 字节 stub 文件
2. 从 vault/raw/original/ 复制原始文件到临时目录
3. 重新运行 vault_ingest.py 导入，12/12 成功
4. 数据无丢失

**为什么犯这个错:**
- PowerShell 的 [string].Replace(char, char) 不接受空字符串做替换
- 以为 Replace([char]12, '') 和 -replace [char]12, '' 等价
- 验证不充分，没检查 $cleaned 变量的异常状态

**教训:**
- PowerShell 字符串清理用 $s -replace [char]12, ''（正则替换，支持空串）
- 不用 $s.Replace([char]12, '')（char 重载，不能接受空串）
- 批量操作前先 dry-run 或用 1 个文件验证

## [12:10] strip('\\"') 转义 Bug — 修复失败 4 次

**发生了什么:**
health_check.py 和 validate_vault.py 中有 strip('\\"')。
在 Python 源码中 '\\"' 运行时产生的是 \"（2 字符：反斜杠+引号）。
但意图是只 strip 引号 "，应该用 strip('"')。

修复过程踩了坑：
1. PowerShell 内联 Python 的嵌套引号导致 SyntaxError
2. 临时脚本中的字符串字面量匹配不对
3. 直到用字节码替换（bytes.replace）才正确命中
4. 前后尝试了 4 种方案才修对

**教训:**
- Python 源码中的转义要在字节级别理解：'\\"' = 4 源字符 → \" 运行时
- 跨语言（PowerShell→Python）的字符串替换优先用 bytes.replace + 硬编码字节
- 文件级别的字符串替换验证要用 read_bytes() 而不是 read_text()

### 新知识
1. **178号文第三十四条**：董事高管权益授予价值不高于授予时薪酬总水平的40%（非171号文的30%）
2. **178号文第三十五条(十三)**："授予时薪酬总水平"=年薪×授予间隔期（一般2年，最高3年），非单年
3. **178号文第二十九条**：每期权益授予间隔期一般2年
4. **国税函〔2009〕461号**：限制性股票应纳税所得额=((登记日市价+解禁日市价)/2-授予价格)×股数
5. 张江高科董事长刘樱2025年年薪127.23万元（来源：2026-03-31披露的2025年报）
6. **227号令(2025修正)**：实际是落实新《公司法》对21部规章的批量修正，内容为删除"监事"相关表述

### 纠正的错误
1. 30%→40%：之前误用171号文30%标准，实际178号文(2020，最新)规定40%
2. 单年→间隔期：薪酬总水平按间隔期年数计算，一般为2年
3. 张江高科适用浦东办事指南：境内预期收益30%（第2条），实际收益上限40%（第4条）
