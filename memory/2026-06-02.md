## 预防规则

### 导入新文章时
1. 先检查 raw/user/ 中是否已有同名 URL（检查 YAML 的 `url` 字段）
2. 如果有相同 URL 的文件，跳过导入
3. 导入后检查文件是否以单 BOM 开头（`ï»¿`），若是双重 BOM 则自动修复

### 日期处理
- 文件名中的 `YYYY-MM-DD` 前缀是**导入日期**，不是发表日期
- 实际发表日期以正文中的 `原创 XXX *YYYY年M月D日*` 为准
- 统计时按正文日期归月，不得使用文件名日期

### 编码规范
- 所有 `.md` 文件统一使用 UTF-8 编码（带 BOM）
- PowerShell 终端显示 UTF-8 文件时中文会乱码（GBK vs UTF-8 冲突），这是正常现象
- 判断文件是否乱码须用 Python 读取验证，不可依赖 PowerShell 的 `cat`/`Get-Content` 输出
